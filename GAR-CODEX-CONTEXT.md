## Engineering workflow

For every mission:

- Provide implementation plan before writing code.
- Wait for approval.
- Implement only approved scope.
- Run tests.
- Run repository checks.
- Commit changes.
- Provide verification report.
- Do not begin the next mission without approval.

## Permanent rules

- Never invent architecture.
- Never implement future sprint functionality.
- Never modify GAR constitutional documents without approval.
- Keep platform packages service-independent.
- Keep Universal Object System under packages/objects.
- Keep business logic out of GAR-SPRINT-0002.
- Preserve architecture-first engineering.
