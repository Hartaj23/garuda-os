# v0.5.0-alpha Release Notes

## Release

- Version: `v0.5.0-alpha`
- Sprint: GAR-SPRINT-0005
- Release theme: Context Foundation
- Preparation status: Prepared pending tag approval
- Recommended tag: `v0.5.0-alpha`

## Summary

GAR-SPRINT-0005 completes the service-independent Context Foundation under `packages.context`.
The release adds platform Context objects, source and scope records, deterministic payloads,
validation integration, descriptive composition and selection contracts, a process-local Context
workspace, certification coverage, and SDK documentation.

## Added

- Universal Context Framework with `UniversalContext`, context types, context states, confidence,
  context metadata and Platform Core inheritance.
- Context Source & Scope Framework with descriptive source records, scope boundaries and
  deterministic metadata payloads.
- Serialization and validation integration for deterministic Context payloads and Platform Core
  validation compatibility.
- Context Composition Contract for descriptive composition type, metadata, composition and contract
  models using opaque context identifiers.
- Context Selection Contract for descriptive selection type, criterion, metadata, request and
  contract models.
- Context Workspace for process-local `UniversalContext` references with exact identifier
  operations.
- Context Foundation platform integration and quality certification suite with a permanent sprint
  certification record.
- Context Foundation SDK documentation, including developer, architecture, API reference, best
  practices, extension and practical examples guides.

## Known Limitations

- `ObjectSerializer.serialize()` emits inherited Platform Core fields and does not include
  Context-specific payload fields under the current Platform Core contract.
- `ContextWorkspace` is process-local and non-persistent.
- Context Composition Contract is descriptive only and does not compose Context objects.
- Context Selection Contract is descriptive only and does not select, search, filter, rank or
  retrieve Context objects.
- Context source identifiers, scope boundaries, composition identifiers and selection criteria are
  opaque records and do not resolve external records.
- The release does not include persistence, databases, file storage, search, ranking, scoring,
  retrieval engines, query engines, semantic lookup, Context Engine behavior, Knowledge Graph,
  Reasoning, AI, REST APIs, frontend features, workflow engines, trading systems or portfolio
  systems.
