# v0.3.0-alpha Release Notes

## Release

- Version: `v0.3.0-alpha`
- Sprint: GAR-SPRINT-0003
- Release theme: Memory Foundation
- Preparation status: Prepared pending tag approval
- Recommended tag: `v0.3.0-alpha`

## Summary

GAR-SPRINT-0003 completes the service-independent Memory Foundation under `packages.memory`.
The release adds platform memory objects, provenance records, deterministic payloads, validation
integration, descriptive index and retrieval contracts, a process-local reference store,
certification, and SDK documentation.

## Added

- Universal Memory Framework with `UniversalMemory`, memory types, confidence, memory metadata and
  Platform Core inheritance.
- Memory Source & Provenance Framework with source descriptors, origins, acquisition methods,
  acquisition channels, provenance references and provenance metadata.
- Serialization and validation integration for deterministic memory payloads and Platform Core
  validation compatibility.
- Memory Index Contract for descriptive index field, metadata, descriptor and contract models.
- Memory Retrieval Contract for descriptive criteria, request, response, status and metadata
  models with opaque memory identifiers only.
- In-Memory Reference Store for process-local `UniversalMemory` references with exact identifier
  operations.
- Memory Foundation certification suite and permanent certification report.
- Memory Foundation SDK documentation, including developer, architecture, API reference, best
  practices, extension and practical examples guides.

## Known Limitations

- `ObjectSerializer.deserialize()` reconstructs Platform Core fields only and does not reconstruct
  memory-specific fields under the current Platform Core contract.
- `MemoryReferenceStore` is process-local and non-persistent.
- Memory Index Contract is descriptive only and does not index memories.
- Memory Retrieval Contract is descriptive only and does not retrieve memories.
- The release does not include persistence, databases, file storage, search, ranking, scoring,
  retrieval engines, vector databases, embeddings, Knowledge Foundation, Context Engine,
  Reasoning, AI, REST APIs, frontend features, workflow engines, trading systems or portfolio
  systems.
