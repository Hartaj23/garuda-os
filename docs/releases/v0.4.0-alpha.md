# v0.4.0-alpha Release Notes

## Release

- Version: `v0.4.0-alpha`
- Sprint: GAR-SPRINT-0004
- Release theme: Knowledge Foundation
- Preparation status: Prepared pending tag approval
- Recommended tag: `v0.4.0-alpha`

## Summary

GAR-SPRINT-0004 completes the service-independent Knowledge Foundation under
`packages.knowledge`. The release adds platform Knowledge objects, evidence and provenance
records, deterministic payloads, validation integration, descriptive classification and query
contracts, a process-local reference store, certification coverage, and SDK documentation.

## Added

- Universal Knowledge Framework with `UniversalKnowledge`, knowledge types, governance states,
  confidence, knowledge metadata and Platform Core inheritance.
- Knowledge Origin, Evidence and Provenance Framework with evidence types, opaque evidence
  references, evidence groups, knowledge origins and provenance records.
- Serialization and validation integration for deterministic Knowledge payloads and Platform Core
  validation compatibility.
- Knowledge Classification Contract for descriptive category, dimension, metadata, descriptor and
  contract models.
- Knowledge Query Contract for descriptive query type, constraint, metadata, query and contract
  models.
- Knowledge Reference Store for process-local `UniversalKnowledge` references with exact
  identifier operations.
- Knowledge Foundation SDK documentation, including developer, architecture, API reference, best
  practices, extension and practical examples guides.

## Known Limitations

- `ObjectSerializer.deserialize()` reconstructs Platform Core fields only and does not reconstruct
  Knowledge-specific fields under the current Platform Core contract.
- `KnowledgeReferenceStore` is process-local and non-persistent.
- Knowledge Classification Contract is descriptive only and does not classify Knowledge.
- Knowledge Query Contract is descriptive only and does not execute, search, filter, rank or
  retrieve Knowledge.
- Evidence references are opaque identifiers and do not resolve external records.
- The release does not include persistence, databases, file storage, search, ranking, scoring,
  retrieval engines, query engines, semantic lookup, Knowledge Graph, Context Engine, Reasoning,
  AI, REST APIs, frontend features, workflow engines, trading systems or portfolio systems.
