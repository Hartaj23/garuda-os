# v0.7.0-alpha Release Notes

## Release

- Version: `v0.7.0-alpha`
- Sprint: GAR-SPRINT-0007
- Release theme: Universal Decision Foundation
- Preparation status: Prepared pending tag approval
- Recommended tag: `v0.7.0-alpha`

## Summary

GAR-SPRINT-0007 completes the service-independent Universal Decision Foundation under
`packages.decision`. The release adds platform Decision objects, descriptive decision inputs,
provenance records, deterministic payloads, validation integration, descriptive strategy and chain
contracts, a process-local Decision workspace, certification coverage, and SDK documentation.

## Added

- Universal Decision Framework with `UniversalDecision`, decision types, decision states, decision
  outcomes, confidence, metadata and Platform Core inheritance.
- Decision Inputs for descriptive opaque references to Memory, Knowledge, Context, Reasoning or
  external records.
- Decision Provenance for descriptive origin records, author metadata, input references and
  deterministic provenance payloads.
- Decision serialization and validation certification for deterministic Decision payloads and
  Platform Core validation compatibility.
- Decision Strategy Contract for descriptive strategy type, metadata, strategy and contract
  models.
- Decision Chain Contract for descriptive chain type, metadata, step reference, chain and contract
  models using opaque decision identifiers.
- Decision Workspace for process-local `UniversalDecision` references with exact identifier
  operations.
- Decision Foundation platform integration and quality certification suite with a permanent sprint
  certification record.
- Universal Decision Foundation SDK documentation, including developer, architecture, API
  reference, best practices, extension and practical examples guides.

## Known Limitations

- `ObjectSerializer.serialize()` emits inherited Platform Core fields and does not include
  Decision-specific payload fields under the current Platform Core contract.
- `DecisionWorkspace` is process-local and non-persistent.
- Decision Inputs and Decision Chain step references are opaque records and do not resolve
  external records.
- Decision Strategy Contract is descriptive only and does not execute strategies.
- Decision Chain Contract is descriptive only and does not execute chains.
- The release does not include decision engines, strategy execution, chain execution, outcome
  computation, reference resolution, provenance evaluation, orchestration, planning, optimization,
  persistence, databases, file storage, search, ranking, scoring, retrieval engines, query engines,
  semantic lookup, AI integration, REST APIs, frontend features, workflow engines, autonomous
  behavior, trading systems or portfolio systems.
