# v0.8.0-alpha Release Notes

## Release

- Version: `v0.8.0-alpha`
- Sprint: GAR-SPRINT-0008
- Release theme: Universal Action Foundation
- Preparation status: Prepared pending tag approval
- Recommended tag: `v0.8.0-alpha`

## Summary

GAR-SPRINT-0008 completes the service-independent Universal Action Foundation under
`packages.action`. The release adds platform Action objects, descriptive action inputs,
provenance records, deterministic payloads, validation integration, descriptive strategy and chain
contracts, a process-local Action workspace, certification coverage, and SDK documentation.

## Added

- Universal Action Framework with `UniversalAction`, action types, action states, action outcomes,
  confidence, metadata and Platform Core inheritance.
- Action primitives for descriptive type, state, outcome, confidence and metadata records.
- Action Inputs for descriptive opaque references to Memory, Knowledge, Context, Reasoning,
  Decision or external records.
- Action Provenance for descriptive origin records, source identifiers, input references and
  deterministic provenance payloads.
- Action serialization and validation certification for deterministic Action payloads and Platform
  Core validation compatibility.
- Action Strategy Contract for descriptive strategy type, metadata, strategy and contract models.
- Action Chain Contract for descriptive chain type, metadata, step reference, chain and contract
  models using opaque action identifiers.
- Action Workspace for process-local `UniversalAction` references with exact identifier operations.
- Action Foundation platform integration and quality certification suite with a permanent sprint
  certification record.
- Universal Action Foundation SDK documentation, including developer, architecture, API reference,
  best practices, extension and practical examples guides.

## Known Limitations

- `ObjectSerializer.serialize()` emits inherited Platform Core fields and does not include
  Action-specific payload fields under the current Platform Core contract.
- `ActionWorkspace` is process-local and non-persistent.
- Action Inputs and Action Chain step references are opaque records and do not resolve external
  records.
- Action Strategy Contract is descriptive only and does not execute strategies.
- Action Chain Contract is descriptive only and does not execute chains.
- The release does not include action engines, strategy execution, chain execution, outcome
  computation, reference resolution, provenance evaluation, scheduling, orchestration,
  optimization, persistence, databases, file storage, search, ranking, scoring, retrieval engines,
  query engines, semantic lookup, AI integration, REST APIs, frontend features, workflow engines,
  autonomous behavior, trading systems or portfolio systems.
