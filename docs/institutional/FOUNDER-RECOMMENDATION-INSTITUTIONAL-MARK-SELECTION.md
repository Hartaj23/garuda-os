Every candidate must be shown in all of the following contexts:

| Context | Specification |
| --- | --- |
| Header scale | 24px |
| Footer scale | 16px |
| Pure black | `#000000` on white |
| On white | `#FFFFFF` ground |
| On the Canonical Page | Applied per [CANONICAL-PAGE-SPECIFICATION-v1.0.md](../../website/brand/CANONICAL-PAGE-SPECIFICATION-v1.0.md) placement rules |
| Print colophon | Representative A4 colophon context |

Every candidate must **survive without color**.

---

## Evaluation Method

Score every candidate against the six criteria established in [Institutional Mark Exploration v1.0](../../website/brand/INSTITUTIONAL-MARK-EXPLORATION-v1.0.md):

1. Acknowledgment, not manufacture
2. Monochrome survival
3. Canonical Page compatibility
4. Token alignment
5. Design Language compliance
6. Institutional character

No intuition-only decisions.

Evidence, again.

---

## Three Institutional Acts

Throughout this entire launch, Project Garuda has consistently separated:

```
Exploration    →    Selection    →    Adoption
```

Most organizations compress those into one meeting.

Garuda has treated them as three distinct institutional acts.

That pattern should not change.

Even after a preferred mark is selected, the institution still pauses before adopting it.

That is not bureaucracy.

That is respect for permanence.

An institution that expects to outlive its founders should choose the symbol that will accompany it for generations with exactly that discipline.

## Proposed Founder Decision (When Issued)

### Founder Decision

**Mission 001 — Phase B**

**Institutional Mark Selection v1.0 Authorization**

### Decision

Institutional Mark Selection v1.0 is hereby authorized.

Authorization is limited to the exploration and evaluation of a small number of institutional mark candidates in accordance with the approved Institutional Mark Exploration v1.0.

The objective is not to create identity.

The objective is to identify the mark that most faithfully acknowledges the identity already established by the institution's visual system.

### Scope

This mission authorizes:

- development of a limited set of institutional mark candidates,
- presentation of each candidate in monochrome,
- evaluation against the approved six institutional criteria,
- application to the Canonical Page Specification,
- application to representative print contexts,
- comparative review.

### Explicitly Not Authorized

This authorization does not permit:

- adoption of a final institutional mark,
- favicon design,
- CSS implementation,
- website implementation,
- public deployment,
- public announcement.

Selection and adoption remain separate Founder decisions.

### Success Criteria

Institutional Mark Selection v1.0 is complete when:

- a limited candidate set has been evaluated,
- every candidate has been assessed against the approved criteria,
- reviewers conclude that one candidate most naturally belongs to the Project Garuda page,
- the institution possesses sufficient evidence to make an informed adoption decision.

The institution shall adopt no mark merely because one has been created.

Only the mark that best serves the institution may proceed to future consideration.
