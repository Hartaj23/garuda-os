## Recommendation — Hosting Environment

**Choose Option 2: an institution-owned domain from day one.**

Not because GitHub Pages is unsuitable as infrastructure.

Because an institution of this significance deserves a public face that reflects the institution — not the hosting provider.

### Candidate domains (for Founder selection)

| Domain | Notes |

## Recommended Platform — Cloudflare Pages

| Criterion | Assessment |
| --- | --- |
| Performance | Excellent global CDN |
| Cost | Free at this scale |
| Integration | GitHub integration; automatic deployments from `master` |
| Domain | Custom domain support; HTTPS by default |
| Lock-in | No vendor lock-in |
| Separation | Clean separation between repository and hosting |

This aligns with Garuda's philosophy:

- **Repository remains canonical.**
- **Hosting remains infrastructure.**
