## Purpose

This plate illustrates the constitutional authority chain that governs every meaningful act of creation within Project Garuda.

The principle is simple:

**Authority always precedes action.**

Every governance transition must derive its legitimacy from an explicitly authorized predecessor.

No authority is assumed.
No authority is inferred.

# Authority Principles

## 1. Authority Flows Downward

Every stage inherits legitimacy from the stage immediately before it.

No stage may create its own authority.

## 2. Drafting Is Not Ratification

Creating a proposal does not make it institutional law.

Authority is created only through explicit ratification.

## 3. Interpretation Is Not Legislation

Architectural records interpret constitutions.
They do not amend them.

## 4. Planning Is Not Authorization

Sprint specifications define implementation.
They do not authorize implementation.

## 5. Engineering Is Not Governance

Engineering faithfully implements approved work.
It does not determine institutional direction.

# The Four Questions

Every participant in Project Garuda should be able to answer four questions before beginning meaningful work.

| Question | Answered By |
| --- | --- |
| Why does this work exist? | Constitutional authority |
| Who authorized it? | Governance record |
| How should it be interpreted? | Architectural record |
| What exactly may be implemented? | Sprint specification |

If any answer is missing, the authority chain is incomplete.

# Why This Matters

Traditional software projects often begin with implementation and document decisions afterward.

Project Garuda reverses that sequence.

Ideas first become legitimate.
Legitimate ideas become architecture.
Approved architecture becomes bounded implementation.
Completed implementation becomes institutional history.

By preserving this order, the institution ensures that every significant artifact can be traced back to an explicit source of authority.

---

# Key Insight

Authority Before Action is not a workflow.
It is an institutional safeguard.

Its purpose is to ensure that creation never outruns legitimacy, interpretation never exceeds authority, and implementation always remains faithful to the institution's constitutional intent.

Every meaningful artifact within Project Garuda exists because an unbroken chain of authority made its existence legitimate.
