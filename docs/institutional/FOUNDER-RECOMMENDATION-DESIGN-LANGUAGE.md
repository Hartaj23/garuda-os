---

## Recommendation

The next artifact should not be colors, typography, or a logo.

There is one missing bridge: **Design Language v1.0**.

We have philosophy.

Next we need principles translated into rules.

---

## Legitimacy Chain

```
Visual Philosophy
        ↓
Design Language
        ↓
Design Tokens
        ↓
Logo
        ↓
Implementation
```

This keeps the same legitimacy chain used throughout Project Garuda.

Typography and color are **tools**.

Design Language is **judgment**.

Garuda has always separated judgment from implementation.

---

## Design Language Scope

Design Language shall answer:

- How should pages breathe?
- What should hierarchy feel like?
- How should diagrams behave?
- What makes something "Garuda"?
- How should information density be managed?
- What should never happen visually?

Only after those answers exist should typography and color be defined.

---

## Creative Direction — The Page as Identity

Garuda's visual identity should not be remembered primarily for its logo.

It should be remembered for its **page**.

Institutions whose documents are instantly recognizable without a logo achieve identity through layout, margins, typography, and restraint.

If someone removed the logo entirely, a page should still unmistakably feel like Project Garuda.

That is the hallmark of an enduring institutional identity.

---

## Institutional Note

We are no longer designing a website.

We are designing the visual behavior of an institution.

One deliberate step.

One earned decision.
