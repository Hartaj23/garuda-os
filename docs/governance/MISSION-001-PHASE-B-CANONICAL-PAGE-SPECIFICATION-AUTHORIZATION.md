## Founder Decision

Canonical Page Specification v1.0 is hereby authorized.

Authorization is limited to defining the canonical structure of a single Project Garuda page.

The purpose of this specification is to demonstrate the complete visual system established through:

- Visual Philosophy v1.0,
- Design Language v1.0,
- Design Tokens v1.0.

The Canonical Page shall serve as the institutional reference from which future website pages, research papers, preservation volumes, presentations, and public publications may derive consistent visual structure.

It is the canonical expression of Project Garuda's visual identity before the introduction of any institutional mark.

---

## Scope

The Canonical Page Specification may define:

- page width,
- reading measure,
- margins,
- spacing rhythm,
- heading hierarchy,
- paragraph rhythm,
- quotations,
- repository references,
- tables,
- diagrams,
- lists,
- navigation placement,
- whitespace behavior,
- footer structure.

The specification shall describe visual behavior rather than implementation technology.

No CSS shall be authored.

No website implementation shall occur.

No production deployment shall occur.

---

## Explicitly Not Authorized

This authorization does not permit:

- logo design,
- favicon,
- illustrations beyond specification examples,
- CSS implementation,
- component implementation,
- website deployment,
- public announcement,
- branding collateral.

Those activities remain subject to future Founder authorization.

---

## Success Criteria

Canonical Page Specification v1.0 is complete when:

- the visual philosophy is observable without explanation,
- the design language is expressed through measurable layout behavior,
- the design tokens are demonstrated consistently,
- the page remains recognizably Project Garuda without requiring a logo,
- reviewers agree that removing the institutional mark would not diminish the page's identity.

---

## Institutional Guidance

Project Garuda's identity shall emerge first from disciplined structure.

Institutional marks should reinforce that identity rather than create it.

The page precedes the mark.

The institution precedes the brand.

The repository remains the authority.

The Canonical Page becomes the institution's first public expression.

---

End of Canonical Page Specification v1.0 Authorization
