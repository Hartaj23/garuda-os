# Architecture Overview

Garuda is an Operating System for Human Intelligence. The foundation is governed by GAR-0001 through GAR-0015.

## Foundation Principles

- Architecture first, code second.
- Every layer has exactly one responsibility.
- Components are modular, observable, explainable, secure, testable, documented and replaceable.
- Applications do not contain business logic.
- Future capabilities must be introduced only through approved sprint scope.

## GAR-SPRINT-0001 Scope

This sprint creates the engineering platform only. Intelligence, trading, memory, knowledge graph, decision engine, broker APIs, authentication and database design are explicitly out of scope.
