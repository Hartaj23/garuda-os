# Development Workflow

Every Garuda implementation follows the approved architecture-first workflow.

1. Understand the intent.
2. Read relevant architecture documents.
3. Confirm sprint authorization.
4. Identify affected contracts, objects and events.
5. Plan tests and documentation.
6. Implement the smallest approved change.
7. Run validation.
8. Update sprint status.
9. Prepare for architecture review.

If architecture is ambiguous, pause implementation and document the ambiguity.
