# Coding Standards

Garuda code must be readable, predictable, consistent, well-named, documented and small in scope.

## Rules

- Prefer clarity before cleverness.
- Keep each module focused on one responsibility.
- Avoid hidden side effects.
- Avoid duplicated responsibilities.
- Keep architecture boundaries explicit.
- Write tests with implementation.
- Update documentation with implementation.

Tool-specific formatting configuration will be added with the backend, frontend and code-quality tasks in GAR-SPRINT-0001.
